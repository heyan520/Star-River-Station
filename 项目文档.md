# Outdoor Travel App 项目文档

## 1. 仓库概览

Outdoor Travel App 是一个基于 Vue 3 + Vite 开发的户外旅行应用，专注于露营地探索、路线规划和户外活动组织。该应用为用户提供露营地信息查询、地图可视化、详情查看等功能，旨在帮助户外爱好者发现和规划露营体验。

**主要功能/亮点：**
- 露营地列表展示与地图可视化
- 多维度筛选功能（类型、区域、等级）
- 露营地详细信息查看
- 路线规划与导航
- 团队活动创建与管理
- 响应式设计，适配移动端
- 集成高德地图 API

**典型应用场景：**
- 户外爱好者寻找新的露营地点
- 家庭规划亲子露营活动
- 团队组织户外活动
- 摄影爱好者发现绝佳取景地

## 2. 目录结构

项目采用典型的 Vue 3 + Vite 项目结构，代码组织清晰。核心业务逻辑位于 `src/components` 目录，包含所有页面组件；路由配置集中在 `src/router` 目录；静态数据存放在 `src/data` 目录。项目使用 Tailwind CSS 进行样式设计，全局样式定义在 `src/style.css` 中。

```plaintext
├── src/                # 源代码目录
│   ├── App.vue         # 应用根组件
│   ├── main.js         # 应用入口文件
│   ├── style.css       # 全局样式文件
│   ├── assets/         # 静态资源目录
│   │   └── 头像.png    # 用户头像图片
│   ├── components/     # 组件目录（包含所有页面）
│   │   ├── Home.vue    # 主布局组件（含底部导航栏）
│   │   ├── HomePage.vue # 首页内容组件
│   │   ├── CampsiteListPage.vue # 露营地列表页
│   │   ├── CampsiteDetailPage.vue # 露营地详情页
│   │   └── ...         # 其他功能页面组件
│   ├── data/           # 数据目录
│   │   └── campsites.js # 露营地数据文件
│   └── router/         # 路由配置目录
│       └── index.js    # 路由配置文件
├── index.html          # HTML 入口文件
├── package.json        # 项目配置与依赖
├── tailwind.config.js  # Tailwind CSS 配置
├── vite.config.js      # Vite 配置文件
└── README.md           # 项目说明文档
```

**主要文件说明：**
- **src/App.vue**: 应用根组件，使用 `<router-view />` 渲染路由内容
- **src/components/Home.vue**: 主布局组件，包含底部导航栏和页面内容区域
- **src/components/CampsiteListPage.vue**: 露营地列表页面，支持筛选和地图展示
- **src/components/CampsiteDetailPage.vue**: 露营地详情页面，展示详细信息和地图定位
- **src/data/campsites.js**: 露营地数据存储，包含多个营地信息和数据获取函数
- **src/router/index.js**: 路由配置文件，定义了应用的路由结构

## 3. 系统架构与主流程

项目采用典型的 Vue 单页应用（SPA）架构，基于组件化和路由系统构建。整体架构清晰，分为以下几个主要层次：

### 架构设计

1. **视图层（组件）**：
   - 使用 Vue 3 的组件系统构建用户界面
   - 采用布局组件（Home.vue）嵌套功能组件的结构
   - 使用 Tailwind CSS 实现样式管理

2. **路由层**：
   - 基于 Vue Router 实现页面导航
   - 采用父路由嵌套子路由的设计，确保底部导航栏全局显示
   - 支持动态路由参数（如 `/campsite/:id`）

3. **数据层**：
   - 静态数据存储在 `src/data` 目录
   - 提供数据获取工具函数（如 `getCampsiteById`）
   - 组件通过 props 和计算属性获取数据

4. **外部集成**：
   - 集成高德地图 API 实现地图功能
   - 使用 Font Awesome 图标库

### 主要业务流程

1. **用户访问流程**：
   - 用户打开应用 → 加载 App.vue → 渲染路由视图 → 加载 Home.vue 布局 → 渲染具体内容组件

2. **露营地浏览流程**：
   - 首页 → 点击露营基地入口 → 进入露营地列表页 → 浏览/筛选露营地 → 点击详情 → 查看露营地详情

3. **导航与页面切换**：
   - 通过底部导航栏切换主要功能模块
   - 通过页面内返回按钮返回上一页
   - 支持详情页返回列表页

## 4. 核心功能模块

### 4.1 露营地列表与筛选

露营地列表页面（CampsiteListPage.vue）提供用户浏览和筛选露营地的功能。主要特点：

- **地图可视化**：集成高德地图，在地图上展示露营地位置
- **多维度筛选**：支持按类型（摄影、亲子、团建等）、区域（主城、东部等）和等级（休闲、综合等）筛选
- **列表展示**：以卡片形式展示露营地基本信息，包括图片、标题、标签和描述
- **地图交互**：支持地图缩放、定位和信息窗口展示

**核心实现**：
- 使用 `filteredCampsites` 计算属性根据筛选条件动态过滤露营地列表
- 通过高德地图 API 动态添加和更新地图标记
- 实现了下拉筛选菜单的显示/隐藏逻辑

### 4.2 露营地详情展示

露营地详情页面（CampsiteDetailPage.vue）展示单个露营地的详细信息：

- **基本信息**：展示露营地名称、位置、评分、价格和容量等基本信息
- **详细描述**：包括营地特色、设施列表和用户评价
- **地图定位**：在地图上精确显示露营地位置
- **交互功能**：支持收藏、分享、VR全景查看等操作
- **返回导航**：提供返回露营地列表的功能

**核心实现**：
- 使用 `getCampsiteById` 函数根据路由参数获取特定露营地数据
- 通过 `watch` 监听 ID 变化，实时更新数据
- 实现地图初始化和标记点设置

### 4.3 底部导航栏

底部导航栏（在 Home.vue 中实现）提供全局的页面导航功能：

- **主要导航项**：首页、商城、附近、我的
- **状态指示**：当前激活页面高亮显示
- **全局显示**：通过路由嵌套确保在所有页面显示

**核心实现**：
- 使用 Vue Router 的 `$route.name` 判断当前路由
- 通过 `:class` 动态添加激活样式
- 采用路由嵌套结构，确保导航栏在所有页面可见

### 4.4 路由系统

路由系统（router/index.js）负责页面之间的导航和组件渲染：

- **布局路由**：使用 Home.vue 作为父路由容器
- **功能路由**：包含首页、露营地列表、详情页等所有功能页面
- **动态路由**：支持通过 ID 访问特定露营地详情

**核心实现**：
- 使用 `createRouter` 和 `createWebHistory` 创建路由实例
- 采用嵌套路由结构，将所有功能页面作为 Home 组件的子路由
- 为动态路由配置 props 传递，方便组件获取参数

## 5. 核心 API/类/函数

### 5.1 数据获取函数

#### getCampsiteById(id)

**功能**：根据 ID 获取指定露营地的数据

**参数**：
- `id`: 露营地 ID（数字或字符串）

**返回值**：匹配的露营地对象，如果未找到则返回第一个露营地

**使用场景**：在露营地详情页中，根据路由参数获取对应的露营地数据

```javascript
// 从路由参数获取 ID 并获取对应露营地数据
const campsiteData = getCampsiteById(this.id);
```

#### getAllCampsites()

**功能**：获取所有露营地的数据列表

**参数**：无

**返回值**：包含所有露营地的数组

**使用场景**：在露营地列表页中获取完整的露营地数据用于展示和筛选

### 5.2 路由相关 API

#### $router.push(path)

**功能**：导航到指定路径的页面

**参数**：
- `path`: 目标路由路径或路由对象

**使用场景**：在底部导航栏切换页面、从列表进入详情页等导航场景

#### $route

**功能**：获取当前路由信息的对象

**主要属性**：
- `name`: 路由名称
- `params`: 路由参数
- `path`: 当前路径

**使用场景**：判断当前页面、获取动态路由参数等

### 5.3 组件生命周期钩子

#### mounted()

**功能**：组件挂载后的生命周期钩子

**使用场景**：在露营地详情页中初始化地图、加载数据等

```javascript
mounted() {
  this.initMap();
  this.loadCampsiteData();
}
```

### 5.4 高德地图相关

#### initMap()

**功能**：初始化高德地图

**实现**：动态加载高德地图 API，创建地图实例，设置中心点和缩放级别

**使用场景**：在露营地列表页和详情页中初始化地图显示

#### updateMapMarkers()

**功能**：更新地图上的标记点

**使用场景**：在筛选条件变化时，更新地图上显示的露营地标记
