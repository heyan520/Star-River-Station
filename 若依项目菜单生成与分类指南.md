# 若依项目菜单生成与分类指南

## 1. 准备工作：检查环境并获取若依项目

### 1.1 环境检查

请确保您的系统已安装以下软件：

| 软件 | 版本要求 | 检查命令 |
|------|----------|----------|
| JDK | 1.8+ | `java -version` |
| MySQL | 8.0+ | `mysql --version` |
| Redis | 5.0+ | `redis-server --version` |
| Node.js | 14.0+ | `node -v` |
| Maven | 3.6+ | `mvn -version` |

### 1.2 获取若依项目

您可以通过以下两种方式获取若依项目：

**方式一：直接克隆代码**
```bash
git clone https://gitee.com/y_project/RuoYi-Vue3.git
```

**方式二：使用Maven创建**
```bash
# 进入工作目录
cd d:\桌面\MouseWithoutBorders\Outdoor

# 使用Maven创建RuoYi项目
mvn archetype:generate -DgroupId=com.outdoor -DartifactId=outdoor-admin -DarchetypeArtifactId=ruoyi-vue3-archetype -DarchetypeVersion=3.8.5 -DinteractiveMode=false
```

## 2. 数据库准备：创建数据库并导入设计表

### 2.1 创建数据库

登录MySQL并创建数据库：

```sql
-- 登录MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE outdoor_admin CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 退出MySQL
quit
```

### 2.2 导入系统表和设计表

1. 导入若依系统表：
```bash
mysql -u root -p outdoor_admin < RuoYi-Vue3/sql/ruoyi-vue3.sql
```

2. 导入我们的户外旅行应用设计表：
```bash
mysql -u root -p outdoor_admin < "d:\桌面\MouseWithoutBorders\Outdoor\整合后数据库设计方案.sql"
```

### 2.3 配置数据库连接

编辑若依项目的数据库配置文件：

```bash
# 进入项目目录
cd RuoYi-Vue3

# 编辑数据库配置文件
notepad src/main/resources/application-druid.yml
```

修改以下配置项：

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driverClassName: com.mysql.cj.jdbc.Driver
    druid:
      url: jdbc:mysql://localhost:3306/outdoor_admin?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
      username: root  # 替换为您的MySQL用户名
      password: 123456  # 替换为您的MySQL密码
      # 其他配置保持不变
```

## 3. 导入数据表到若依系统

### 3.1 启动若依项目

1. 启动Redis服务：
```bash
redis-server.exe
```

2. 启动后端服务：
```bash
cd RuoYi-Vue3
mvn clean install -DskipTests
java -jar target/ruoyi-admin.jar
```

3. 启动前端服务：
```bash
cd RuoYi-Vue3/ruoyi-ui
npm install
npm run dev
```

4. 访问若依系统：
   - 地址：`http://localhost:80`
   - 账号：admin
   - 密码：admin123

### 3.2 登录并导入数据表

1. 登录若依系统后，进入【系统管理】→【代码生成】页面

2. 点击【导入】按钮，选择需要导入的表

3. 勾选我们设计的户外旅行应用相关表（以`outdoor_`开头）

4. 点击【确定】按钮完成导入

## 4. 配置若依系统代码生成参数

### 4.1 查看导入的表

在【代码生成】页面可以看到已导入的表列表。点击表名可以查看表的详细信息和配置参数。

### 4.2 配置生成参数

以路线表（`outdoor_route`）为例，点击表名进入配置页面，修改以下参数：

| 参数名称 | 参数值 | 说明 |
|---------|--------|------|
| 作者 | admin | 代码作者 |
| 生成模块 | outdoor | 业务模块名 |
| 生成业务名 | route | 业务功能名 |
| 生成功能名 | 路线管理 | 功能显示名称 |
| 生成菜单名 | 路线管理 | 菜单显示名称 |

### 4.3 配置字段属性

在【字段信息】标签页中，可以配置每个字段的属性：

| 字段 | 配置项 | 说明 |
|------|--------|------|
| route_name | 字段备注 | 路线名称 |
| route_name | 查询类型 | 模糊查询 |
| route_name | 显示类型 | 文本框 |
| difficulty_level | 显示类型 | 下拉框 |
| difficulty_level | 字典类型 | route_level |

## 5. 使用代码生成工具生成业务代码

### 5.1 生成代码

1. 在【代码生成】页面，勾选需要生成代码的表
2. 点击【生成代码】按钮
3. 系统会自动生成代码包并下载到本地

### 5.2 代码结构说明

生成的代码包包含以下内容：

```
├── java/           # 后端Java代码
│   ├── controller/ # 控制器
│   ├── service/    # 服务层
│   ├── mapper/     # 数据访问层
│   └── domain/     # 实体类
├── resources/      # 资源文件
│   └── mapper/     # MyBatis映射文件
├── vue/            # 前端Vue代码
│   └── outdoor/    # 业务模块
└── menu.sql        # 菜单SQL脚本
```

## 6. 导入生成的代码到项目中

### 6.1 导入后端代码

1. 解压生成的代码包
2. 将`java`目录下的代码复制到项目的相应目录：
   - `com.outdoor.outdoor.*` → `src/main/java/com/outdoor/outdoor/`
3. 将`resources/mapper`目录下的XML文件复制到：
   - `*.xml` → `src/main/resources/mapper/outdoor/`

### 6.2 导入前端代码

将`vue`目录下的代码复制到前端项目的相应目录：
- `outdoor/*` → `ruoyi-ui/src/views/outdoor/`

### 6.3 刷新项目

1. 后端项目：刷新Maven项目，确保代码编译通过
2. 前端项目：执行`npm run build`重新构建前端代码

## 7. 配置菜单结构和分类

### 7.1 创建菜单分类

在若依系统中，进入【系统管理】→【菜单管理】页面，创建菜单分类：

1. 点击【新增】按钮，创建一级菜单：
   - 菜单名称：户外旅行管理
   - 菜单类型：目录
   - 路由地址：outdoor
   - 图标：el-icon-map-location
   - 排序：100
   - 权限标识：outdoor:manage

### 7.2 导入生成的菜单

可以通过以下两种方式导入菜单：

**方式一：执行SQL脚本**

1. 打开生成代码包中的`menu.sql`文件
2. 登录MySQL执行该脚本：
   ```bash
   mysql -u root -p outdoor_admin < menu.sql
   ```

**方式二：手动创建菜单**

1. 在【菜单管理】页面，点击【新增】按钮
2. 填写菜单信息：
   - 菜单名称：路线管理
   - 父菜单：户外旅行管理
   - 菜单类型：菜单
   - 路由地址：outdoor/route
   - 权限标识：outdoor:route:list
   - 图标：el-icon-guide
   - 排序：1

3. 点击【确定】完成菜单创建

### 7.3 配置菜单权限

为角色分配菜单权限：

1. 进入【系统管理】→【角色管理】页面
2. 选择需要配置的角色（如管理员）
3. 点击【权限设置】按钮
4. 在权限树中勾选需要的菜单权限
5. 点击【确定】保存配置

## 11. 常见问题解决

### 11.1 菜单不显示

**问题描述**：登录系统后，左侧菜单栏中不显示新创建的菜单

**解决方法**：
1. 检查菜单是否已正确创建
2. 检查用户角色是否已分配该菜单的权限
3. 清除浏览器缓存并重新登录

### 11.2 页面访问报错

**问题描述**：点击菜单后，页面显示404或500错误

**解决方法**：
1. 检查前端路由配置是否正确
2. 检查后端接口是否已正确部署
3. 查看浏览器控制台和后端日志，定位具体错误

### 11.3 权限验证失败

**问题描述**：访问页面或执行操作时提示"没有权限"

**解决方法**：
1. 检查用户角色是否拥有该功能的权限
2. 检查接口上的`@PreAuthorize`注解配置是否正确
3. 检查菜单的权限标识是否与接口权限一致
